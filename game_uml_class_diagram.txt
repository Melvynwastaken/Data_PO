RELATIONSHIP LEGEND:
────────────────────
◆ Composition (strong ownership)
● Aggregation (weak ownership)
▲ Inheritance/Realization
─► Implementation (interface)
* One to Many (*)
1 One to One (1)


KEY REQUIREMENTS MET:
──────────────────────
✓ 6+ Classes: Game, Character, Player, Enemy, Boss, Weapon, Item, Inventory, Level
✓ 1 Abstract Class: Character (with 2+ inheritors: Player, Enemy)
✓ 1 Interface: Damageable (implemented by Character and Boss)
✓ All Classes have Relations:
  - Game ──► Level, Player, Enemy[], Weapon, Item[]
  - Character ──► Weapon, Inventory
  - Player & Enemy inherit from Character
  - Boss inherits from Enemy
  - Inventory contains Items
  - Level manages Items and Enemies


RELATIONAL BREAKDOWN:
─────────────────────
1-to-1 Relationships:
  • Game to Player (1:1)
  • Game to Level (1:1)

1-to-Many Relationships:
  • Game to Enemy (1:*)
  • Level to Item (1:*)

Many-to-Many Relationships:
  • Inventory to Item (implicitly through collection)

Abstract Class Inheritance:
  • Character (abstract) ◄── Player, Enemy
  • Enemy ◄── Boss (extended enemy with special abilities)

Interface Implementation:
  • Damageable ◄── Character, Boss
